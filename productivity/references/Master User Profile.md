# Core Identity

I'm Alexander Thompson, an information designer who turns complexity into clarity people can actually use. I think in systems, patterns, and semantics: I break problems into components and prefer practical, reusable solutions that feel coherent and robust. I'm pivoting into entrepreneurship under real financial pressure, so I need decisive action and fast feedback loops, not endless tuning.

### Core Principles

- Clarity over cleverness and simplicity over complexity
- Progress over perfection
- Follow "Do it right the first time"
- Value semantics, intuition, and practicality
- Systems exist to buy back time and freedom

### What I Want to Be Doing

High-leverage systems work: turning complexity into clear, repeatable outcomes. Data pipelines, documentation, information architecture, automation, and tools that scale.

### Tech I Work In

- **Data & Analytics:** Data engineering, analysis, dashboards, and practical applied math.
- **AI & Automation:** Using AI tools as strategic partners for planning, drafting, coding support, and lightweight automations.
- **Information Design:** Technical docs, information architecture, data documentation, and structured explanations.
- **Creative Tools:** Design and prototyping for brands and experiences using tools like Adobe Suite, Figma, and 3D/engine tools.
- **Web & Apps:** Simple, maintainable web/app architectures across presentation, logic, and data layers.

### Keep for Myself or Delegate

- Video editing: creative control for my channels, not client revision cycles
- Graphic design: cohesive brand systems, not scattered one-offs
- Deep programming grinds: I prefer architecture over marathons
- Advanced academic math: I favor intuitive, applied problem-solving

### Hard Avoids

- IT troubleshooting / help desk / "the IT guy" role
- Low-autonomy, repetitive technical work
- Any role defined by support technician expectations
