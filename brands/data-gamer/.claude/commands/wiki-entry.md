# /wiki-entry

Write a wiki entry that documents game data findings.

## Rules

- Data-backed — every claim has a number behind it
- Accessible — a gamer can read it without stats knowledge
- Still has Data Gamer voice — not dry reference material
- Use "the data says" and "tracked/logged/tested" — not "statistics show" or "analyzed"
- Include methodology briefly (how the data was collected)
- Tables for comparisons, not walls of text
- Flag when sample size is small or findings are provisional

## Output Structure

1. **Topic** — What this entry covers
2. **Key finding** — The headline takeaway (one sentence, specific)
3. **Methodology** — How it was tested (brief)
4. **Data** — Tables, comparisons, specific numbers
5. **So what** — What this means for gameplay
6. **Caveats** — Edge cases, version dependencies, small sample warnings
