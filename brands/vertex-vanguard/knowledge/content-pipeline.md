# Content Pipeline

How Vertex Vanguard content moves from idea to published piece to repurposed assets.

## The Pipeline

### 1. Idea

Capture the moment something breaks, surprises you, or clicks. The best VV content comes from real problems encountered during actual builds — not from brainstorming "what should I teach."

Sources:
- A bug or broken export you just spent hours fixing
- A framework that finally made a complex topic click
- A build milestone worth sharing (BTS post)
- A question someone asked that deserved a longer answer

### 2. Outline

Match the idea to a content pattern (from `brand/samples.md`):

| If the idea is... | Use this pattern |
|---|---|
| Something broke and you fixed it | Problem-Fix Tutorial (`/tutorial-script`) |
| A complex topic you can simplify | Framework Distillation (`/framework-distill`) |
| An in-progress build update | Behind-the-Scenes (`/bts-post`) |

Outline the key beats. For Problem-Fix: what broke, why, the fix, the principle. For Framework: the topic, the compression, the examples. Keep it to bullet points — don't write prose yet.

### 3. Draft

Write the full script or post using the appropriate command. The draft should:
- Follow the content pattern structure exactly
- Include all settings and numbers (no placeholders)
- Have the personal struggle moment baked in
- End with the principle or takeaway

For video content (YouTube tutorials, BTS):
- Record screen capture alongside voiceover
- Show the actual project — not a clean demo project
- Capture the real UI, real settings, real results

For written content (blog posts, documentation):
- Screenshots of actual settings and results
- Annotated images where menu paths matter

Key principles:
- Cut the throat-clearing — get to the problem fast
- Keep the messy/honest moments — don't polish them out
- Add timestamps for video content at each major section

Primary platforms:
- YouTube (tutorials, BTS videos)
- Documentation site (written tutorials, frameworks)

### 7. Repurpose

Every long-form piece should produce shorter assets. This is where VV content strategy differs from other brands — the repurposed pieces aren't just clips, they're standalone micro-lessons.

| Source | Repurposed Into |
|---|---|
| Tutorial video | Short-form clip of the key fix (30-60s) |
| Tutorial video | Written tutorial on documentation site |
| Tutorial video | Social post with the principle as the hook |
| Framework post | Social carousel walking through each piece |
| BTS video | Social post with the messy/surprising moment |

## How VV Content Differs

| Other brands | Vertex Vanguard |
|---|---|
| Polish the result, hide the process | The process IS the content |
| "Here's what to do" | "Here's what went wrong and why this fixes it" |
| Theory first, application later | Problem first, principle extracted after |
| Generic advice | Exact settings and numbers |
| One format per idea | Every idea repurposed across formats |

## Metrics

Track these to know if VV content is reaching the right audience:

- **Watch time / read time** — are people staying through the principle section, or dropping off after the fix?
- **Comments mentioning "I had this exact problem"** — signal that you're hitting real pain points
- **Returning viewers/readers** — are people coming back for the next tutorial?
- **Search traffic** — are people finding VV content when they search for specific problems?
